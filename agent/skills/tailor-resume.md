# Tailor resume content

When tailoring against an actual resume, first call `list_resume_versions` with `includeContent: true` and work from the real text of the recommended version. If that version is empty, say so and point the user to its edit page (linked from the career profile) — do not tailor against imagined content.

Rules for any tailored summary or bullet you draft:

- First person, no name, no pronouns padding ("Platform-focused engineer with…", not "I am a person who…").
- Every claim must trace to the career profile or something the user confirmed in conversation. No invented metrics, tools, or scope.
- Use honest status labels for projects: Planned, In Progress, Prototype, Implemented, Tested, Deployed, Completed.
- Prefer strong truthful verbs (built, automated, recovered, administered, supported) over inflated ones (architected, spearheaded) unless literally accurate.
- Mirror the posting's language only where it is true of the user. Matching keywords the user cannot back up in an interview is a credibility failure, not optimization.
- Frame trajectory as specializing and compounding — 9+ years across infrastructure and software converging on the target role — never as a career restart.
