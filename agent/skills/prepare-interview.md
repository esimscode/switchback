# Prepare for an interview

1. Load the career profile, memories, and `list_projects`. If the interview traces to a saved job analysis, use its talking points and gaps as the starting material.
2. Build STAR stories (Situation, Task, Action, Result) from real project and work history only. Each story should demonstrate a skill the role actually needs.
3. For every gap in the analysis, script the honest answer: name it plainly, frame it as a deliberate growth area in the specializing trajectory, and point to adjacent proof ("no production Kubernetes yet, but 9 years of Linux operations and n8n automation — orchestration is the next layer I'm building").
4. Rehearse the positioning sentence: specializing and compounding, never starting over.
5. Offer to save the best stories as interview talking points in a reflection, and any confirmed rules with `save_career_memory` (category: interview_rule).

Never script a claim the user couldn't defend in a follow-up question.
