# Draft a LinkedIn post

Voice: builder sharing real work — direct, specific, warm, zero hype. No "thrilled to announce", no "game-changer", no emoji walls, no engagement-bait questions unless the user asks.

Shape:
1. Open with the concrete thing ("I've been building X — a Y that does Z.").
2. 2-4 short lines on what's interesting: a decision, a problem hit, something learned. Specifics beat adjectives.
3. Honest status — "in progress", "first working version", "shipped for one client". The status label on the project record caps the claim.
4. Optional close: what's next, or a link (GitHub/portfolio) when the project record has one.

Rules:
- Load the career profile first; the post should quietly reinforce the positioning (platform, cloud, automation, security trajectory).
- Never imply production scale, team leadership, or completion that isn't in the record.
- Under 1300 characters unless asked otherwise. Line breaks between thoughts. At most 3 hashtags, only if natural.
